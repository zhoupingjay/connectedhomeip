Detailed Description

This file defines default compile-time configuration constants for the CHIP Time Services profile.

Macro Definition Documentation

◆ CHIP_CONFIG_TIME

#define CHIP_CONFIG_TIME   1

1 (Default) If chip Time Services is needed/enabled. 0 If chip Time Services is not needed. Content of implementation files would be compiled out.

◆ CHIP_CONFIG_TIME_CLIENT_CONNECTION_FOR_SERVICE

#define CHIP_CONFIG_TIME_CLIENT_CONNECTION_FOR_SERVICE   1

Disabled: 0, Enabled: 1. If enabled, the client provides an API to go through a chip connection for time sync. If disabled, contact to cloud service must be fed to the client through chip::Profiles::Time::TimeSyncNode::SyncWithNodes and the transport would be UDP.

◆ CHIP_CONFIG_TIME_CLIENT_FABRIC_LOCAL_DISCOVERY

#define CHIP_CONFIG_TIME_CLIENT_FABRIC_LOCAL_DISCOVERY   1

Disabled: 0, Enabled: 1. If enabled, the client would use UDP multicast to automatically discover available time servers and coordinators in the fabric. If disabled, contact information must be fed to the client through chip::Profiles::Time::TimeSyncNode::SyncWithNodes.

◆ CHIP_CONFIG_TIME_CLIENT_MAX_NUM_CONTACTS

#define CHIP_CONFIG_TIME_CLIENT_MAX_NUM_CONTACTS   4

This only applies to Time Sync Client/Coordinator roles. This is the 'list of known Time Coordinators and Servers' in design spec. Number of unicast contacts a Client/Coordinator can remember.

◆ CHIP_CONFIG_TIME_CLIENT_MAX_RTT_USEC

#define CHIP_CONFIG_TIME_CLIENT_MAX_RTT_USEC   (10 * 1000000LL)

This only applies to Time Sync Client/Coordinator roles. This is the 'max round trip time for usable response' in design spec. Number of usec for the longest round trip time a Client/Coordinator would consider a response as valid/useful. Default is 10 seconds.

◆ CHIP_CONFIG_TIME_CLIENT_MIN_OFFSET_FROM_SERVER_USEC

#define CHIP_CONFIG_TIME_CLIENT_MIN_OFFSET_FROM_SERVER_USEC   (2 * 1000000LL)

This only applies to Time Sync Client/Coordinator roles. This is the 'min_time_offset_from_service' in design spec. If the absolute value of calculated correction from a Time Sync Server is smaller than this number of usec, the correction would be ignored. Default is 2 seconds.

◆ CHIP_CONFIG_TIME_CLIENT_MINIMUM_DISCOVERY_PERIOD_MSEC

#define CHIP_CONFIG_TIME_CLIENT_MINIMUM_DISCOVERY_PERIOD_MSEC   (30 * 60 * 1000)

This only applies to Time Sync Client/Coordinator roles. This is the variable 'timer_minimum_discovery_period' in design spec. Default number of msec for the Client/Coordinator to perform another multicast discover operation, when there is some communication error, if not specified in the init call. It is not recommended to perform discover operation faster than the default value. Default is 30 minutes.

◆ CHIP_CONFIG_TIME_CLIENT_NOMINAL_DISCOVERY_PERIOD_MSEC

Default number of msec for the Client/Coordinator to perform another multicast discover operation, when there is no communication error, if not specified in the init code. Default is 12 hours.

◆ CHIP_CONFIG_TIME_CLIENT_REASONABLE_TIME_SINCE_LAST_SYNC_MIN

#define CHIP_CONFIG_TIME_CLIENT_REASONABLE_TIME_SINCE_LAST_SYNC_MIN   120

This only applies to Time Sync Client/Coordinator roles. This is 'time since last sync for reliable response' in design spec. If a Server/Coordinator indicates that its system time has only been synced to some other reliable source longer than this number minutes ago, it would be counted as a stale/unreliable source. Default is 2 hours.
